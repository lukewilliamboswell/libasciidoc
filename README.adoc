= Libasciidoc

image:https://img.shields.io/badge/License-Apache%202.0-blue.svg["License", link="https://opensource.org/licenses/Apache-2.0"]

Libasciidoc is an open source Go library that converts Asciidoc content into HTML or DOCX.

This is a fork of https://github.com/bytesparadise/libasciidoc[bytesparadise/libasciidoc] with DOCX output support and modernised build tooling.

== Installation

    $ go install github.com/lukewilliamboswell/libasciidoc/cmd/libasciidoc@latest

== Usage

=== HTML output (default)

    $ libasciidoc -s content.adoc
    $ libasciidoc -o - content.adoc          # output to STDOUT
    $ libasciidoc -o output.html content.adoc # explicit output file

=== DOCX output

    $ libasciidoc -b docx content.adoc          # produces content.docx
    $ libasciidoc -b docx -o - content.adoc     # output to STDOUT
    $ libasciidoc -b docx -o out.docx content.adoc

=== All options

    $ libasciidoc --help

== Output Formats

Using `-b` (or `--backend`) the following formats are supported:

* `html5` (also `html`) -- this is the default
* `xhtml5` (also `xhtml`)
* `docx` -- Microsoft Word (.docx) output

== Supported Syntax

Although it does not support the full Asciidoc/Asciidoctor syntax, Libasciidoc provides the following features:

* Title and Sections level 1 to 6
* Document authors and revision
* Attribute declaration and substitution
* Paragraphs and admonition paragraphs
* Delimited Blocks (fenced, listing, example, comment, quoted, sidebar, verse, open)
* Source code highlighting via https://github.com/alecthomas/chroma[Chroma] (use `chroma` or `pygments` as the `source-highlighter`)
* Literal blocks
* Quoted text (bold, italic, monospace, marked, superscript, subscript) and backslash escaping
* Typographic quotes and symbols (copyright, registered, trademark)
* Passthrough (single plus, triple plus, `pass:[]`, `pass:q[]`)
* External links (`https://`, `http://`, `ftp://`, `irc://`, `mailto:`)
* Inline and block images
* Icons (font, graphic, admonition blocks and inline)
* Element attributes (ID, link, title, role, short-hand)
* Ordered, unordered, labeled, and checklist lists
* Tables (header line, multi-line cells, top-level styles)
* Horizontal rules and page breaks
* Table of contents
* YAML front-matter

See also link:LIMITATIONS.adoc[known limitations].

=== Syntax Highlighting

When enabled, syntax highlighting in `[source]` blocks uses https://github.com/alecthomas/chroma[Chroma].
The default class prefix is `tok-`, overridable with the `chroma-class-prefix` attribute:

[source]
----
:chroma-class-prefix: myprefix-
----

Or from the command line:

    $ libasciidoc -o - -a chroma-class-prefix=myprefix- mydoc.adoc

== Code Integration

Libasciidoc provides two functions for programmatic use:

1. Convert from an `io.Reader`:

    Convert(r io.Reader, output io.Writer, config *configuration.Configuration) (types.Metadata, error)

2. Convert a file by name:

    ConvertFile(output io.Writer, config *configuration.Configuration) (types.Metadata, error)

The returned `types.Metadata` contains the document title, table of contents, and other attributes.
All options are passed via the `config` parameter.

=== Macro Definition

Define custom macros with `configuration.WithMacroTemplate()`:

```go
var tmplStr = `<span>Example: {{.Value}}{{.Attributes.GetAsString "suffix"}}</span>`
var t = template.New("example")
var tmpl = template.Must(t.Parse(tmplStr))

output := &strings.Builder{}
content := strings.NewReader(`example::hello world[suffix=!!!!!]`)
libasciidoc.Convert(content, output,
    configuration.NewConfiguration(
        configuration.WithMacroTemplate(tmpl.Name(), tmpl)))
```

== Development

=== Prerequisites

* Go 1.24+
* https://github.com/mna/pigeon[pigeon] (for parser generation, installed via `go install`)

=== Building

    $ go build ./cmd/libasciidoc

=== Testing

    $ go test ./...

=== Parser Generation

The parser is generated from `pkg/parser/parser.peg` using pigeon:

    $ go generate ./...

=== Linting

    $ golangci-lint run
